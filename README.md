# 琳喵喵后处理

`琳喵喵后处理` 是一个运行在 SillyTavern 前端里的第三方扩展。

它的作用是：

- 读取当前聊天窗口最后一条消息
- 按你的范围设置截取正文或指定标签内容
- 叠加你选中的模板内容，整理出最终发送文本
- 把整理后的文本发给当前酒馆模型，或发给你填写的自定义 API
- 收到结果后，再决定是否替换当前聊天最后一条消息

这个扩展适合下面几类场景：

- 润色最后一条消息
- 只处理消息中的某一段标签内容
- 用一组固定模板反复加工文本
- 先看结果、再手动确认是否替换原消息
- 需要在确认前对回复内容做最后编辑

## 当前版本功能概览

当前版本主要包含 5 组能力：

1. `设置范围`
   - 支持整条处理
   - 支持只截取某对标签之间的内容
   - 支持保留标签
   - 支持仅替换标签内文本
2. `模板预设`
   - 支持多预设、多模板
   - 支持勾选模板、拖拽排序、编辑、删除
   - 支持预设切换、新增、重命名、删除
   - 支持模板 / 预设导入导出
3. `截取与发送`
   - 支持手动触发
   - 支持手动发送
   - 支持自动触发
   - 支持跳过确认
   - 支持停止流程
4. `回复确认框`
   - 支持直接编辑 AI 回复
   - 支持确认替换
   - 支持重来
   - 支持带反馈重新生成（肘击）
5. `悬浮窗`
   - 支持开启 / 关闭
   - 支持点击、长按、双击绑定功能
   - 支持拖拽位置
   - 支持自定义宽高


说明：

- `settings.html` 加载失败时，插件会回退到内置备用面板
- `zhaopan/linmm.webp` 缺失时，悬浮窗图片无法正常显示

加载成功后，底部栏会出现一个 `H` 按钮，标题为“琳喵喵后处理”。点击它即可打开面板。

## 面板结构

插件面板目前分为三列：

- 左侧：功能菜单
- 中间：当前功能的设置详情
- 右侧：截取与输出框

左侧当前菜单包括：

- `设置范围`
- `模板预设`
- `截取与发送`
- `api链接`
- `悬浮窗设置`

右侧的“截取与输出”框是可编辑的。
也就是说，插件自动整理出来的内容，你可以继续手动修改，最终发送内容以这里的文字为准。

## 30 秒快速上手

第一次使用时，建议按这个最短路径体验：

1. 点击底部 `H` 按钮打开面板
2. 在 `设置范围` 中先把开始标签和结束标签都留空
3. 在 `模板预设` 中勾选一个你想用的模板
4. 切到 `截取与发送`
5. 点击 `手动触发`
6. 看右侧“截取与输出”框是否是你想发给模型的内容
7. 点击 `手动发送`
8. 在弹出的确认框中：
   - 直接点 `确认`，或
   - 先改文字再点 `确认`

这样你就完成了一次最基本的“截取 -> 处理 -> 发送 -> 替换”流程。

## 新手推荐上手流程

如果你准备正式开始用，建议按下面流程配置一次：

### 第一步：确认处理范围

进入 `设置范围`：

- 如果你想处理最后一条消息的全文：开始标签和结束标签都留空
- 如果你只想处理某段内容：填写开始标签和结束标签，例如 `text` 和 `text`
- 如果你需要保留原始标签包裹：勾选 `保留标签`
- 如果你只想替换标签内部文字：在勾选 `保留标签` 后，再开启 `仅替换标签内`

### 第二步：准备模板

进入 `模板预设`：

- 勾选你要启用的模板
- 已勾选的模板会按列表顺序叠加到截取的内容前面
- 如果顺序不对，可以长按模板按钮后拖拽调整顺序
- 如果没有合适模板，可以新增或编辑模板
- 模板就是预设条目,整个模板其实就是预设功能,把你的要求写到模板中以便ai输出需要的功能哦

### 第三步：生成发送内容

进入 `截取与发送`：

- 点击 `手动触发`
- 插件会读取当前聊天最后一条消息
- 再根据“范围设置 + 已勾选模板”生成最终输出
- 结果会写到右侧“截取与输出”框

这一步完成后，建议先检查右侧文本是否符合预期。

### 第四步：发送并确认替换

确认右侧文本没问题后：

- 点击 `手动发送`
- 默认会弹出 `AI 回复确认` 框
- 你可以直接编辑回复内容
- 最后点击 `确认`，把结果替换到当前聊天最后一条消息

### 第五步：按需要启用自动流程

如果你不想每次都手动打开确认流程，可以在 `截取与发送` 中继续调整：

- `开启自动触发`：当前聊天生成结束后，自动跑一轮插件流程
- `跳过确认`：收到回复后直接替换，不再弹出确认框

## 详细功能说明

### 1. 设置范围

这一栏决定“最后一条消息里，哪些内容会被取出来处理”。

可用项：

- `开始标签`
- `结束标签`
- `保留标签`
- `仅替换标签内`

行为说明：

- 两个标签都留空时，默认处理最后一条消息全文
- 只填写一个标签或标签不完整时，会被视为无效范围
- 勾选 `保留标签` 后，输出中会保留标签本身
- 勾选 `仅替换标签内` 后，确认替换时会优先尝试只替换匹配标签内的文本

如果标签匹配失败、数量不一致，或无法安全替换：

- 插件不会强行改写原消息结构
- 无法替换的内容会被追加到右侧“截取与输出”框末尾

适合场景：

- 只处理 `<text>...</text>` 中的正文
- 只重写消息中的某个结构化字段
- 保留角色卡生成格式，只修改正文内容

### 2. 模板预设

这一栏用来管理一组可复用模板。

当前支持：

- 点击模板按钮切换选中 / 取消选中
- 长按模板按钮拖拽排序
- 编辑模板
- 删除模板
- 新增预设
- 重命名预设
- 删除当前预设
- 切换当前预设

补充说明：

- 已选模板会按当前列表顺序，从上到下叠加到输出文本前面
- 新建预设时会自动带上默认模板
- 删除模板时会弹出确认界面
- 删除模板界面里的 `本次使用不再提示` 只对当前酒馆会话生效，重启后恢复
- 删除预设时也会先弹出确认框，确认后才真正删除

### 3. 模板导入导出

当前支持两种导出：

- `导出当前预设`
- `导出当前选定模板`

导入时插件会自动识别内容类型：

- 导入预设：会新增一个预设
- 导入模板：会先确认，再把模板追加到当前预设

导入规则：

- 只接受 `.json` 文件
- 名称重复时会自动补 `(数字)`
- 导入到当前预设的模板，默认是未选中状态

### 4. 截取与发送

这是最常用的一栏。

它包含 3 组内容：

#### 4.1 手动流程按钮

- `手动触发`
  - 读取当前聊天最后一条消息
  - 根据范围设置做截取
  - 根据已选模板生成最终输出
  - 把结果写到右侧输出框
- `手动发送`
  - 把右侧输出框内容发送给模型
- `停止流程`
  - 停止当前插件流程

#### 4.2 流程开关

- `开启自动触发`
  - 这是按聊天单独保存的开关
  - 当前聊天打开后，刷新页面仍会保留
  - 切换到别的聊天时，会读取那个聊天自己的状态
- `跳过确认`
  - 开启后，收到结果会直接替换最后一条消息
  - 关闭后，仍然走确认框流程

#### 4.3 输出框

右侧 `截取与输出` 文本框：

- 会显示最终准备发送的内容
- 可以手动继续编辑
- 插件真正发送的就是这里的文字

### 5. 回复确认框

默认情况下，发送完成后会弹出 `AI 回复确认` 框。

当前确认框支持：

- 直接编辑 AI 回复内容
- `确认`
- `重来`
- `肘击`
- 关闭确认框

各按钮说明：

- `确认`
  - 使用你当前看到的文字进行替换
- `重来`
  - 用上一次完全相同的发送内容重新生成一次
- `肘击`
  - 进入反馈输入界面
  - 输入你不满意的地方后点击 `提交`
  - 插件会把这段反馈拼到上一次发送内容前面，再重新生成
- `返回`
  - 从反馈界面回到确认界面
  - 当前确认框内容不会因此丢失

### 6. 自动触发

开启 `自动触发` 后，插件会在当前聊天生成结束后自动执行一轮：

1. 截取最后一条消息
2. 套用范围设置和模板
3. 生成新的发送文本
4. 自动发给模型
5. 根据你的设置：
   - 进入确认框，或
   - 直接替换最后一条消息

注意：

- 自动触发只对当前聊天生效
- 这个状态保存在聊天元数据里，不是全局统一开关
- 如果你在自动流程执行中点了 `停止流程`，后续结果会按插件逻辑中止，不会继续强行替换消息

### 7. API 链接

默认情况下，插件可以直接复用酒馆当前模型。

你也可以切换到自定义接口，手动填写：

- API 地址
- API 密钥
- 模型名称

如果接口支持模型列表，还可以点 `获取模型` 获取后直接选择。

当前可调项包括：

- 模型温度
- Top-P
- Top-k
- 存在惩罚
- 频率惩罚
- 流式输出
- 停止字符
- 模型来源

补充说明：

- 在 `与酒馆相同` 模式下，这些参数会尽量临时覆盖当前生成参数
- 在 `自定义` 模式下，实际生效项取决于当前接口兼容性
- 停止字符会在收到回复后再做本地截断处理

### 8. 悬浮窗

在 `悬浮窗设置` 里可以控制悬浮窗的显示和触发行为。

支持项：

- `开启悬浮窗`
- `悬浮窗调整大小`
- `点击触发的功能`
- `长按触发的功能`
- `双击触发的功能`

当前可选动作包括：

- 未设置
- 打开设置范围
- 打开模板预设
- 打开截取与发送
- 打开api链接
- 打开悬浮窗设置
- 运行手动触发
- 运行手动发送
- 开关自动触发
- 运行停止流程
- 打开截取与输出框

交互说明：

- 长按不移动：执行长按绑定的功能
- 长按后拖动：移动悬浮窗位置
- 支持点击、双击、长按三套独立动作
- 拖拽后的位置会保存
- 尺寸设置也会保存

默认尺寸：

- 桌面端约 `88 x 88`
- 移动端约 `72 x 72`

## 推荐使用方式

### 场景 1：润色最后一条消息

1. `设置范围` 留空
2. 在 `模板预设` 中勾选润色类模板
3. 点击 `手动触发`
4. 检查右侧输出框
5. 点击 `手动发送`
6. 在确认框中确认或微调后确认

### 场景 2：只处理标签中的正文

1. 在 `设置范围` 中填写开始标签和结束标签，例如 `text`
2. 视需要开启 `保留标签` 或 `仅替换标签内`
3. 点击 `手动触发`
4. 检查右侧输出框
5. 点击 `手动发送`

### 场景 3：固定使用一套常用模板

1. 在 `模板预设` 中新建一个预设
2. 添加并整理模板顺序
3. 勾选常用模板
4. 以后切换到这个预设即可直接复用

### 场景 4：希望自动跑完整个流程

1. 配好范围和模板
2. 在 `截取与发送` 中开启 `自动触发`
3. 如果你仍想人工把关，就保持 `跳过确认` 关闭
4. 如果你希望收到回复后直接替换，就开启 `跳过确认`

## 设置保存说明

当前项目中，大部分设置会保存在扩展设置里，包括：

- 范围设置
- 模板预设与模板内容
- API 设置
- 悬浮窗开关、位置、尺寸、动作
- 跳过确认

`自动触发` 的保存方式不同：

- 它按聊天单独记录
- 保存到当前聊天的元数据中
- 所以不同聊天可以有不同开关状态

## 注意事项

- 插件处理的是“当前聊天窗口最后一条消息”
- 右侧输出框可手动编辑，最终发送内容以这里的文本为准
- 默认不会直接覆盖原消息，而是先进入确认框；只有开启 `跳过确认` 后才会直接替换
- 如果标签匹配失败，插件不会强行破坏原消息结构
- 如果使用自定义 API，请自行确认接口兼容性和密钥安全
- 当前仓库没有自动化测试，真实验证方式仍然是在 SillyTavern 宿主里手动测试

## 适合谁用

如果你经常需要：

- 对最后一条消息做润色、总结、抽取要点
- 只处理消息中的某一段标签内容
- 反复复用一组固定模板
- 想在替换前先看结果、再决定是否覆盖
- 想在确认框里继续“重来”或带反馈重试

那这个插件会比较适合你。
