# 琳喵喵后处理

这是一个运行在 SillyTavern 前端里的第三方扩展，用来对当前聊天窗口最后一条消息做二次处理。

它的典型用法是：

1. 截取当前聊天最后一条消息的全部内容，或者只截取指定标签中的内容。
2. 把已选中的模板预设内容自动加到截取结果前面。
3. 将整理后的文本发送给当前酒馆模型，或发送到你填写的自定义 API。
4. 收到回复后先弹出确认框，你可以先修改回复，再决定是否替换当前聊天的最后一条消息。

## 安装方式

把本插件文件夹整体放到 SillyTavern 的第三方扩展目录中。

常见做法是把整个目录复制到 SillyTavern 的 `public/scripts/extensions/third-party/` 下，然后刷新或重载扩展。当前代码会根据实际加载到的 `index.js` 路径自动解析扩展目录，所以文件夹名称不必强制叫 `my-topbar-test`，但目录内部相对结构要保持不变。

建议把下面这些文件和目录一起完整保留：

- `manifest.json`
- `index.js`
- `style.css`
- `settings.html`
- `zhaopan/linmm.webp`

说明：`settings.html` 加载失败时，插件会回退到内置备用面板；但 `zhaopan/linmm.webp` 缺失时，悬浮窗图片本身就无法正常显示。

加载成功后，顶部栏会出现一个 `H` 按钮，标题为“琳喵喵后处理”。点击它即可打开面板。

## 简单上手流程

如果你是第一次用，建议按下面流程走一遍：

1. 打开酒馆后，点击顶部的 `H` 按钮。
2. 先进入“设置范围”：
   - 如果你想处理整条最后消息，开始标签和结束标签都留空。
   - 如果你只想处理某段标签内容，就填写开始标签和结束标签，例如 `text` 和 `text`。
3. 进入“模板预设”：
   - 选中你想启用的模板。
   - 选中的模板会按列表顺序叠加到输出内容最前面。
4. 进入“截取与发送”：
   - 点击“手动触发”，右侧“截取与输出”框会自动填入处理后的文本。
   - 如有需要，可以直接在右侧输出框里手动继续修改。
5. 点击“手动发送”：
   - 插件会把右侧输出框内容发送给模型。
   - 收到结果后会弹出“AI 回复确认”框，你可以先编辑回复，再点击“确认”替换当前聊天最后一条消息。

如果你不想替换，可以直接关闭确认框。

## 功能说明

### 1. 设置范围

- `开始标签` / `结束标签`：用于从最后一条消息中提取指定片段。
- 两个都留空时，默认处理最后一条消息的全文。
- `保留标签`：打开后，输出中会保留开始标签和结束标签本身。
- `仅替换标签内`：只在勾选“保留标签”后可用。开启后，确认替换消息时会优先尝试只替换匹配标签中的内容；如果标签匹配失败或数量对不上，插件不会强行替换，而是把无法替换的内容追加到右侧输出框末尾。

适合场景：

- 只处理 `<text>...</text>` 中的正文。
- 只重写某个结构化字段，而不是整个消息。

### 2. 模板预设

“模板预设”用于管理一组可复用的模板。

- 一个预设里可以放多个模板。
- 点击模板按钮是选定或取消选定。
- 长按模板按钮可以拖拽调整顺序。
- 已选中的模板会按当前列表从上到下的顺序，依次加到输出内容最前面。
- 每个模板都可以编辑、删除。
- 删除模板时会弹出确认框；勾选“本次使用不再提示”后，本次酒馆会话中删除模板不再重复确认，重启后恢复。

预设区支持：

- 切换当前预设
- 新增预设
- 重命名预设
- 删除当前预设

新建预设时会自动带上 3 个默认模板。

### 3. 模板导入导出

导出时可以选择：

- `导出当前预设`
- `导出当前选定模板`

导入时插件会自动识别文件是“预设”还是“模板”：

- 导入预设：会新增一个预设。
- 导入模板：会先弹出确认框，确认后再追加到当前预设的模板列表末尾。
- 如果名称重复，会自动在名称后加上 `(数字)`。

导入模板时，导入后的模板默认是未选中状态。

### 4. 截取与发送

这一栏主要做两件事：

- `手动触发`：读取当前聊天最后一条消息，并根据“设置范围”和“模板预设”生成最终输出。
- `手动发送`：把右侧输出框里的内容发送给模型。

右侧“截取与输出”框是可编辑的，所以你可以：

- 先自动生成，再手动改字。
- 不走模板，直接把它当作临时编辑区使用。

`停止流程` 的作用：

- 如果当前正在执行自动流程，会优先停止自动流程。
- 如果当前正在发送手动请求，会停止这次插件流程；底层请求不一定会被立刻物理中断，但后续结果会按当前逻辑忽略。
- 如果手动流程的确认框已经弹出，也可以先确认是否关闭确认框，再停止本次流程。
- 如果自动流程的确认框已经弹出，停止后会直接关闭确认框，并把当前内容按插件逻辑保留到右侧输出框。

### 5. API 链接

默认情况下，插件可以直接使用酒馆当前模型。

如果你想用自定义接口，可以在“api链接”里切换到 `自定义`，再填写：

- API 地址
- API 密钥
- 模型名称

如果接口支持模型列表，还可以点击“获取模型”帮助选择模型名。

在“与酒馆相同”模式下，这些参数会尽量临时覆盖当前生成参数；在“自定义”模式下，实际生效项取决于当前代码里发送到接口的字段。一般来说，温度、Top-P、存在惩罚、频率惩罚、模型名会参与请求；停止字符会在收到回复后再做本地截断，是否流式也以当前实现为准。

### 6. 自动触发

开启“自动触发”后，插件会在当前聊天里监听生成结束事件，并自动执行一轮：

1. 截取最后一条消息
2. 按当前“设置范围”和已选模板重新生成输出
3. 发给模型
4. 弹出确认框

需要注意：

- 自动触发是按“当前聊天”生效的，切换聊天后会自动关闭。
- 如果你在自动流程中执行“停止流程”，当前结果会按插件逻辑中止，不会强行继续替换消息。

### 7. 悬浮窗

在“悬浮窗设置”里可以控制悬浮窗的显示和触发行为。

- `开启悬浮窗`：勾选后显示悬浮窗，不勾选时隐藏。
- `悬浮窗调整大小`：可以分别填写“长度”和“宽度”，单位都是 `px`。留空时会使用默认值；当前默认值在桌面端是 `88 x 88`，在移动端是 `72 x 72`。
- `点击触发的功能` / `长按触发的功能` / `双击触发的功能`：三项共用同一套动作列表，可以分别配置为打开面板页、运行一次功能、开关自动触发或执行停止流程。
- 悬浮窗图片使用扩展目录下的 `zhaopan/linmm.webp`。
- 当前长按判定时间约为 `0.32` 秒。
- 长按后不移动松开时,会执行“长按触发的功能”；只有长按后继续拖拽，并且位移超过约 `12px`，才会进入位移模式。
- 如果在长按成立前就移动超过约 `12px`，这次长按不会成立，也不会进入拖拽。
- 双击判定窗口约为 `0.26` 秒。
- 拖拽后的悬浮窗位置，以及悬浮窗尺寸设置，都会保存在插件设置里，刷新或重启后会恢复到上次保存的值。

当前可选动作列表为：

- 未设置
- 打开设置范围
- 打开模板预设
- 打开截取与发送
- 打开api链接
- 打开自动触发
- 打开悬浮窗设置
- 运行手动触发
- 运行手动发送
- 开关自动触发
- 运行停止流程
- 打开截取与输出框

## 推荐使用方式

### 场景 1：润色最后一条消息

1. 范围留空。
2. 在模板预设中选中“润色”类模板。
3. 点击“手动触发”。
4. 检查右侧输出框。
5. 点击“手动发送”。
6. 在确认框里确认替换。

### 场景 2：只处理标签中的正文

1. 在“设置范围”中填写开始标签和结束标签，例如 `text`。
2. 如有需要，打开“保留标签”或“仅替换标签内”。
3. 再执行“手动触发”与“手动发送”。

### 场景 3：维护一套常用模板

1. 在“模板预设”里新建一个预设。
2. 添加多个模板并排好顺序。
3. 需要时直接切换预设，不必每次重新勾选。
4. 用导出保存整套预设，便于备份或分享。

## 注意事项

- 插件处理的是“当前聊天窗口最后一条消息”。
- 发送后的结果不会直接覆盖消息，而是先进入确认框；你可以修改后再确认。
- 导入模板到当前预设前会先确认一次；停止手动流程时，如果确认框已打开，也会再确认是否关闭；停止自动流程时会直接关闭确认框，并把内容保留到右侧输出框。
- 右侧输出框内容可手动编辑，最终发送内容以这里的文本为准。
- 如果使用自定义 API，请自行确认接口兼容性和密钥安全。
- 插件设置会保存在 SillyTavern 的扩展设置中，刷新后会保留；这也包括悬浮窗的开关、动作、位置和尺寸。

## 适合谁用

如果你经常需要：

- 对最后一条消息做润色、总结、抽取要点
- 只处理消息里某一段标签内容
- 反复复用一组固定提示词模板
- 手动确认 AI 结果后再替换原消息

那这个插件会比较适合你。
